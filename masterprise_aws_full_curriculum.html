Execution Model

12-Week Build Sequence

Each phase ends with a runnable, deployable AgriChain increment. Never start the next phase on a broken foundation.

Phase 1 — Foundations (W1-W3)

Linux, Git, networking, Python/Bash, Docker. End state: containerized AgriChain running locally with docker-compose.

LinuxGitDocker
Phase 2 — AWS Platform (W4-W7)

IAM, VPC, Terraform, ECS Fargate, RDS, EKS. End state: AgriChain API running on EKS with managed database in AWS.

TerraformECSEKSRDS
Phase 3 — Data Engineering (W8-W10)

EventBridge, SQS, Kinesis, Glue, Athena, CI/CD, GitOps. End state: event-driven harvest pipeline feeding an S3 lakehouse.

KinesisGlueAthenaArgoCD
Phase 4 — Production Ready (W11-W12)

CloudWatch, X-Ray, GuardDuty, WAF, dbt, QuickSight. End state: fully observable, secure platform with an analytics dashboard.
