Bangkok E-Commerce Batch Pipeline
Build a full ELT pipeline: raw orders → cleaned warehouse → star schema. Build on both AWS and GCP and compare the developer experience.
Starter 3–4 weeks

Concepts Applied

Cloud Services Side-by-Side
AWS Stack
S3 Bronze — raw files partitioned by date prefix
Glue Crawler Auto-crawl S3 → populate Glue Data Catalog with schema
Athena Ad-hoc SQL on Bronze without loading to Redshift
Redshift Silver + Gold via dbt; COPY from S3 for bulk ingest

GCP Stack

dbt Core Same dbt models — just swap profiles.yml to BigQuery adapter
Cloud Composer Managed Airflow — DAGs in GCS, GKE-backed environment
AWS ArchitectureCSV/API → S3 (Bronze) → Glue Crawler → Catalog → Athena (ad-hoc) → Redshift COPY → dbt Silver / Gold → QuickSight / Metabase
GCP ArchitectureCSV/API → GCS (Bronze) → BQ External Tables → BigQuery Silver / Gold → dbt models → Looker Studio / Metabase
⚖ Key Comparison After Building
Redshift requires cluster management, VACUUM/ANALYZE maintenance, and WLM queue config. Query performance depends heavily on sort/distribution keys you choose upfront. BigQuery is fully serverless — no cluster, no keys, but you pay per TB scanned. Ask: which matters more for a startup — operational simplicity or cost predictability at 100TB scale? Write your answer down as your first ADR.
★ Challenge Extension
Simulate a bug where the pipeline used the wrong THB→USD exchange rate for 3 days. Write a backfill script that reprocesses only affected date partitions — on both S3/Redshift and GCS/BigQuery. Observe how each cloud handles partial partition overwrites. This forces real understanding of idempotency in both environments.

Streaming Services Side-by-Side
AWS Streaming Stack
Kinesis Streams Shard-based, ordered within shard, 7-day max retention
KDA (Flink SQL) Managed Flink — windowed aggregations on Kinesis without cluster ops
Kinesis Firehose Dead-simple delivery: stream → S3/Redshift, no code required
MSK (Kafka) Managed Kafka — use if you want Kafka-native ecosystem on AWS
GCP Streaming Stack

AWS ArchitecturePython Producer (boto3) → Kinesis Data Streams → KDA Flink SQL (5-min windows) → Kinesis Firehose → S3 → Redshift COPY → Grafana (Redshift)
GCP ArchitecturePython Producer (pubsub) → Pub/Sub Topic → Dataflow (Apache Beam) → 5-min window aggregations → BigQuery streaming insert → Grafana (BigQuery)
⚖ Key Comparison After Building
Kinesis requires upfront shard capacity planning — more events means more shards, more cost, and ordering is only guaranteed per shard. Pub/Sub is fully serverless, scales automatically, but does NOT guarantee ordering by default (need Ordering Keys explicitly). Which is simpler to operate for a 3-person team? Which handles a 10× traffic spike more gracefully? Also compare: Kinesis shard-hour pricing vs Pub/Sub message-volume pricing — which is cheaper for your event volume?
★ Challenge Extension

A downstream consumer writes this to a "surge_zones" table. Build on both clouds and compare how each handles multi-topic fan-out. Document the cost difference: Kinesis charges by shard-hours, Pub/Sub by message volume — which is cheaper for a system sending ~10K surge events per day?

Managed Spark: EMR vs Dataproc
AWS Lakehouse Stack
S3 + Delta Lake S3A connector — careful config needed for ACID guarantees over S3
EMR EC2-based Spark — full control, slower startup (~5–10 min), spot support
EMR Serverless Submit Spark jobs without cluster management — pay-per-vCPU-hour
Glue 4.0 Supports Delta Lake natively — simpler than EMR for most batch transforms
Lake Formation Table/column-level governance over your Delta lakehouse
GCP Lakehouse Stack
GCS + Delta Lake GCS connector — better-integrated with GCP auth model than S3A
Dataproc Managed Spark — fast ~90s startup, GCS/BigQuery native connectors
Dataproc Serverless Submit Spark batch without cluster — same concept, faster cold start
BigLake Query Delta/Iceberg tables on GCS directly from BigQuery
Dataplex Catalog, lineage, and quality across the whole lakehouse in one place
AWS LakehouseSources → S3 Bronze (Delta format) → EMR / Glue 4.0 (PySpark) → S3 Silver (MERGE, typed) → dbt + EMR Serverless → S3 Gold (ACID, time-travel) → Lake Formation permissions → Athena / Redshift Spectrum
GCP LakehouseSources → GCS Bronze (Delta format) → Dataproc / Serverless (PySpark) → GCS Silver (MERGE, typed) → dbt + Dataproc → GCS Gold (ACID, time-travel) → BigLake + Dataplex → BigQuery External → Looker
⚖ Key Comparison After Building
EMR gives you maximum control but significant operational overhead — cluster config, bootstrap actions, spot interruption handling, ~5–10 min startup. Dataproc is leaner with ~90s startup, making ephemeral clusters practical for daily jobs. For serverless: EMR Serverless vs Dataproc Serverless — benchmark both on the same 10GB dataset. Which finishes first? Which costs less? Which was easier to debug when a job failed?

⚖ Final ADR — AWS vs GCP Recommendation
After building everything on both, write a 1-page Architecture Decision Record answering: for a Thai fintech startup needing to move fast with a 3-person data team, which cloud do you recommend?

Consider: AWS — largest ecosystem, strongest Thai e-commerce/logistics presence, bigger local hiring pool, more service options but more choices to manage. GCP — BigQuery's serverless model eliminates a whole class of operational problems, Datastream is the best CDC tool, better for analytics-heavy teams. There's no universally right answer. The quality of your reasoning is the deliverable.

For the Thai Job Market — Which Cloud Matters Where
Fintech & Banking
SCB, KBank, TTB, Ascend Money — heavily GCP and Azure. BigQuery + Datastream is popular. PDPA compliance, audit trails, and real-time fraud are the top requirements. GCP depth is a strong differentiator here.
E-Commerce & Logistics
Lazada, Flash Express, Kerry, LINE MAN — AWS-heavy due to tech ancestry. S3 + EMR + Redshift stack common. Airflow + dbt is dominant. AWS certification carries real weight in these companies.
Consulting & MNC
Accenture, EY, Deloitte Thailand — clients use both. Being fluent in both clouds and able to recommend one over the other with clear reasoning is the most valuable skill. Your ADRs from Project 5 are your interview material.
